Gemini Enterprise Agent Platform でノーコード エージェントを構築する

はじめに

Gemini Enterprise Agent Platform は、エンタープライズ グレードのエージェントを構築、デプロイ、管理できる Google Cloud のサービスです。 Gemini Enterprise Agent Platform は 4 つのサービス カテゴリに分けることができ、それぞれに本番環境でエージェントを有効にするためのプロダクトとサービスのスイートが含まれています。以下はその 4 つのカテゴリです。

  1. 構築（Build）: エージェントをゼロから構築できるプロダクトとサービスのスイート。このサービスを使用すると、複数の方法でエージェントを構築できます。完全にノーコードでエージェントを構築することもできますし、要件に応じてコードを記述してエージェントを構築することもできます。このカテゴリには、基盤モデル、Agent Development Kit (ADK)、Agent to Agent Protocol (A2A)、Universal Commerce Protocol (UCP) などの主要なプロダクトとサービスが含まれています。
  2. 拡張（Scale）: 効率的な短期および長期のメモリ管理のための Agent Sessions と Agent Memory Bank を備えた Agent Runtime を介して、エージェントをデプロイおよび拡張します。
  3. ガバナンス（Govern）: Gemini Enterprise Agent Platform には、本番環境のエージェントを監視および保護するために設計された堅牢なガバナンス スイートが含まれています。Agent Identity、Model Armor、Agent Anomaly Detection などの主要なサービスは、セキュリティと運用の整合性を維持するための不可欠なフレームワークを提供します。
  4. 最適化（Optimize）: このレイヤーは、Agent Observability、Agent Optimizer、Agent Evaluation などの主要なサービスを活用して、エージェントの品質とパフォーマンスを向上させるためのさまざまな技術を提供します。

ノーコード エージェントの構築

GUI を使用して、コードをまったく記述せずに、Gemini Enterprise Agent Platform でシンプルなエージェントを作成してデプロイできます。このブログでは、Gemini Enterprise Agent Platform でシンプルなレストラン検索エージェントを作成してデプロイします。

Agent Platform の Studio を使用してノーコード エージェントを構築できます。次の手順に従ってください。

  1. Google Cloud コンソールに移動します。検索バーに「Agent Platform」と入力すると、Agent Platform サービスに移動します。そこに移動したら、サイドメニューから Studio をクリックします。次の画面が表示されます。

図 2: Agent Platform Studio

  1. サイドバーの Agents をクリックします。
  2. 「Create Agent」ボタンをクリックします。Agent Designer インターフェースが表示されます。

    図 3: Agent Creator インターフェース
  3. 「My Agent」バーをクリックすると、エージェントの詳細が表示されます。必要に応じてエージェントの詳細を変更します。以下の例をご覧ください。次の情報を追加できます。
    Name: エージェントの名前
    Description: エージェントの説明
    Instruction: これには、エージェントが実行することになっている詳細な指示が含まれます
    Model: エージェントに使用させる基盤モデルの選択



    図 4: エージェント詳細インターフェース
  4. エージェントにツールを使用させることもできます。ツールを追加するには、サイドバーの [Tools] セクションにある「+」をクリックします。

    図 5: エージェント ツール インターフェース
  5. 現在、次のツールがサポートされています。Google 検索を実行するための Google Search Tools、コンテンツに URL のコンテンツを追加するための URL Context ツールを追加できます。MCP ツールを使用してエージェントを拡張することもできます。プライベート データ ソースからの情報を提供する Vertex AI Search データストアと、カスタム MCP サーバーを追加できます。

    図 6: エージェントへのツールの追加
  6. サブエージェントの別のレイヤーを追加して、タスクをサブエージェントに委任することもできます。サブエージェントを追加するには、エージェントをクリックして「+」ボタンをクリックします。

    図 7: サブエージェントの追加
  7. トップレベルのエージェントにレストランを検索するタスクを委任し、チェッカー エージェントを使用して結果を確認するように依頼することで、レストラン検索エージェントを作成できます。


    図 8: エージェント構造
  8. この場合、各エージェントに次の指示を追加できます。


Restaurant Agent

Instructions:
Execute the following Flow

  1. Use the Restaurant Finder Agent sub agent to find the list of Restaurants
  2. Once you get the response, use Restaurant Checker Agent to get the confidence score.
  3. Display both the Confidence Score and the Restaurant information to the user.

Tools: None

Restaurant Finder Agent

Instructions:
   Find the restaurants and provide information about them in the following format,

          1. Restaurant Name and Address

          2. Ratings

          3. Menu items and price

    eg.

    User query:

    Find me a restaurant near Shibuya Station

    Expected Response:

    Shibuya Ramen, Shibuya, 5 mins from shibuya station

        Rated 4.5/5

    Available menu:

         Tonkotsu Ramen: 800 Yen

         Miso Ramen: 900 Yen

Once the result is retrieved ask the Restaurant Checker Agent to check the response

Tools: Google Search Tool

Restaurant Checker Agent

Instructions:
   Look at the user prompt and see the response is correct,

    The response should be a list of restaurants in the following format

  1.  Restaurant Name and Address
  2.  Ratings
  3. Menu items and price

Look at the results and compute the Confidence Score with max score being 5, eg. 4.5/5 and send back.

Tools: Google Search Tool
