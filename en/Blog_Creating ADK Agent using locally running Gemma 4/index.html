Gemma Model Comparison


Before selecting the model I wanted to check the performance of the model in terms of Tokens/s and do a quick smoke test. In this blog I will be creating an agent that gives me information about birds so I asked a very obscure question about birds.

What is the difference between a Vega Gull and a Herring Gull?

This is by no way meant to be a comprehensive test but if I met a person who could tell the difference between a Vega Gull and a Herring Gull, I would consider that person very knowledgeable. This was the basis of my comparison.

Model 1: Gemma 4 26B  (4-bit quantized)

This model is a 4-bit quantized version of Gemma 4 26B parameters. The model requires 15GB memory+20% of working space. From my specs you can see it is going to be very tight.

Model:gemma4:26b (4-bit quantized model)
Run Command:  ollama run gemma4:26b --verbose

Prompt: What is the difference between a Vega Gull and a Herring Gull

Satisfactory Answer: No.

Time to First Token: 8.146811s

Prompt Eval Rate: 4.30 tokens/s

Eval Rate:17.55 tokens/s

It started off okay, it got the taxonomy of Larus vegas correctly but got all the details wrong. It had potential but it just missed the mark.The performance was quite slow because the model could not fit in the VRAM. It utilized my system RAM too. The data needed to move between VRAM and System RAM made the model perform quite slow however I felt the speed was acceptable for my task.

Model 2: Gemma 4 12B (4-bit quantized)

This model is a 4-bit quantized version of Gemma 4 12B parameters. The model requires around 8GB+20% working space. It should do better than the 26B model in terms of token generation.

Model:gemma4:12b (4-bit quantized model)
Run Command:  ollama run gemma4:12b --verbose

Prompt: What is the difference between a Vega Gull and a Herring Gull

Satisfactory Answer: No

Time to First Token:  911.771ms

Prompt Eval Rate: 39.48 tokens/s

Eval Rate :7.45 tokens/s

This model completely missed the mark when it came to the answer to my question. It thought Vega Gull was a European Gull whereas it's actually an East Asian Gull and it also missed the taxonomy. Time to first token was very fast (less than a second), prompt eval rate was also very fast. However the Eval Rate was not as fast as I had expected.I ran this a few times with very similar results. This is something I need to investigate further.

Considering the 26B Model seemed slightly more accurate without too much performance penalty, I decided to pick this as my model for my Agent. However, out of curiosity I decided to give the 31B model a try as well just to see if it would get right, even if the performance would not be great.

Model 3: Gemma 4 31B (4-bit quantized)

This model is a 4-bit quantized version of Gemma 4 31b parameters. The model requires around 20GB memory+20% working space.

Model:gemma4-31b (4-bit quantized model)
Run Command:  ollama run batiai/gemma4:31b  --verbose

Prompt: What is the difference between a Vega Gull and a Herring Gull

Satisfactory Answer: Yes

Time to First Token:  3.900597s

Prompt Eval Rate: 7.43 tokens/s

Eval Rate : 1.21 tokens/s

Gemma 4 31B model got it perfectly correct! Although it was really crawling in my machine at around 1 token/s. Running 31B for my agent would be painfully slow so I decided to stick with the 26B model. I think Gemma 4 31B is a very capable model and I would have loved to run it!

Setting up ADK (Agent Development Kit)

In order to use ADK, you need to install the Python ADK package and the preferred way to do so is by setting up a Python virtual environment. Once you set up the python virtual environment, install ADK using pip as follows.

pip install google-adk

Create and Run an Agent locally

  1. Create a project folder eg. gemmaagent
  2. Open the folder in your favorite text editor (eg. Antigravity)
  3. Run Gemma 4 locally using following command,

     ollama run gemma4:26b
  4. Once you run Gemma using ollama, ollama automatically opens up an API server to interact with Gemma 4.
  5. Create a folder called agent1 inside your project folder and create a file inside agent1 folder with the filename agent.py with the following code.

import os

import sys

import asyncio

import urllib.request

import json

# Ensure UTF-8 output formatting on Windows to prevent Unicode encoding errors

if os.name == 'nt':

    os.environ["PYTHONUTF8"] = "1"

    try:

        sys.stdout.reconfigure(encoding='utf-8')

        sys.stderr.reconfigure(encoding='utf-8')

    except AttributeError:

        pass  # Standard streams might not support reconfiguring in all execution contexts

# Import Google ADK and GenAI types

try:

    from google.adk import Agent

    from google.adk.runners import Runner

    from google.adk.sessions.sqlite_session_service import SqliteSessionService

    from google.adk.models.lite_llm import LiteLlm

    from google.genai import types

except ImportError:

    print("[CRITICAL] Google ADK package not found. Please run:")

    print("  pip install google-adk litellm")

    sys.exit(1)

# Monkeypatch Pydantic serialization bug in LiteLlm to exclude llm_client

if hasattr(LiteLlm, "model_fields") and "llm_client" in LiteLlm.model_fields:

    LiteLlm.model_fields["llm_client"].exclude = True

# Constants

OLLAMA_API_BASE = "http://localhost:11434"

DEFAULT_MODEL = "ollama/gemma4:26b"

DB_PATH = "sessions.db"

SESSION_ID = "default_gemma4_session"

USER_ID = "local_developer"

# Configure Environment for LiteLLM

os.environ["OLLAMA_API_BASE"] = OLLAMA_API_BASE

model_connector = LiteLlm(model=DEFAULT_MODEL)

root_agent = Agent(

    name="local_gemma_assistant",

    model=model_connector,

    instruction=(

        "You are a helpful, direct, and concise programming assistant. "

        "You are powered by the local Gemma-4 model via the Google Agent Development Kit (ADK). "

        "Keep your responses focused and technically accurate."

    )

)



  1. As you can see from the code, a LiteLlm object connects to the locally running Gemma 4 and lets you use Gemma 4 as your ADK model.
  2. Use the terminal to go to your project folder and enter the following command to run the Agent.

    adk web

  3. The server should start in port 8000

  4. In the browser enter the following URL to connect to ADK server.

    http://localhost:8000
  5. Select your agent and test it.
